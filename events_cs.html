AInigma Decode Battle Byte (Hackathon)

AInigma

Details

  • The participants are given 90 minutes.
  • The programming language is fixed.
  • The students are given one hour (the first 60 minutes) to study the programming language and get familiar with it to a very minimalistic level.
  • Internet access is given to them only for the first 60 minutes and not for the next 30 minutes. (Mobile phones will be restricted for those 30 minutes)
  • Then a problem is given to them considering the amount of familiarity a beginner can gain in 60 minutes to a new programming language. The problem is to be solved using the fixed language.

Requirements

Participants are supposed to have laptops.

Team Size

It is an individual event

Motive

To gain a friendly environmental attitude towards learning and not competing.

Constraint

This event is only for the first years (2018 batch).

Decode Battle

Details

There are two problems in this event:

  • A computational or an algorithmic problem where the answer is a number, or a string.
  • This problem is encrypted or encoded with the answer obtained above. The problem will be given as a file. The contents of the file will be encoded with the above obtained answer. The participants are supposed to decode the problem and then solve it.
This will cover their logical thinking skills for the first problem and their applicational skills via the second problem. There is no language constraint for this event. Even assembly is accepted (if they are comfortable).

Team Size

It is an individual event

Deadline

The event will run for two hours and cannot be brought down to less than 90 minutes upon negotiation for event scheduling.

Constraint

This event is open to everyone in any branch of the Institute.

Byte (Hackathon)

(In collaboration with Zerone Club)

Description

The participants are given 3 to 4 hours, so the problem statement is going to be a difficult task. There is only problem statement given to all teams. Internet access is allowed throughout the event.

Team Size

2 members

Criteria for Judging

  • Lines of code
  • Binary size
  • Processing time
  • Memory Utilization (not CPU utilization) For example, if they’re implementing a web server; The server shouldn’t crash given 1 million HTTP requests to the end point.

Example Scenario

If a submission ‘A’ takes 10 secs to compute the task and takes 500 mb of memory space. And another submission ‘B’ takes 15 secs to compute the task and takes 150 MB of memory space, then ‘B’ is the winner. 350 MB of extra memory is costlier than 5 seconds of extra time.
